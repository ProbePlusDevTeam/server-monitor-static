Data Retention & Scalability

Control how long your logs and metrics are stored. Built on reliable, scalable storage so Server Monitor grows with your infrastructure — no rewrites needed.

What It Does

📅

Configurable Retention Periods

Choose how long logs and metrics are stored — from 7 days (Starter) up to 90 days (Enterprise). Balance storage cost with audit needs.

📈

Scales with Your Fleet

Add more hosts and containers without performance degradation. The storage layer is built to handle high-volume metric and log ingestion.

⚡

Fast Historical Retrieval

Optimised for quick queries — charts and log lookups remain fast even when querying weeks of data across many hosts.

Pros & Cons

Pros

  • ✓ Longer retention for audit trails and compliance review
  • ✓ No manual purge management — automatic rolling retention
  • ✓ Storage scales as you add more hosts without extra setup
  • ✓ On-premises deployments have full control over storage sizing
  • ✓ Fast retrieval even with large historical datasets

Cons & Considerations

  • ! Retention caps at 90 days on Enterprise cloud — longer retention requires on-premises
  • ! Very high-volume log producers may require on-prem deployment for storage control
  • ! No data export API yet — data lives within Server Monitor (export planned)
