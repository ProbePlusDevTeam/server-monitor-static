Feature

Security & Access Control (RBAC)

Secure login with JWT. Four roles — SuperAdmin, Admin, Operator, and Viewer — each controlling exactly what a user can see and do across your infrastructure.

What It Does

🔐

JWT Authentication

Secure, token-based login for all API and dashboard access. Sessions are managed and expire safely.

👥

Four-Tier RBAC

SuperAdmin → full control. Admin → manage users & settings. Operator → run container actions. Viewer → read-only access.

📋

Audit Trail

Every container action and config change is logged with who did it and when — essential for compliance and incident review.

Pros & Cons

Pros

  • ✓ Prevents unauthorized container operations
  • ✓ Complies with least-privilege security principles
  • ✓ Audit trail supports compliance requirements (SOC 2, ISO 27001)
  • ✓ Gives MSPs safe client access without exposing other clients
  • ✓ Built-in from day one — no third-party IAM setup needed

Cons & Considerations

  • ! Four fixed roles — custom role permissions not yet supported
  • ! SSO / SAML integration only on Enterprise plan
  • ! No MFA / 2FA support yet (on roadmap)
  • ! Group-based RBAC (team-level permissions) planned for a future release
