Feature

Agent Configuration & Control

Set heartbeat intervals, refresh rates, and alert thresholds per environment. Push config changes to agents instantly — no service restarts or SSH required.

What It Does

📡

Dynamic Configuration Push

Update agent settings from the dashboard and have them applied immediately — no need to touch the server or restart the agent.

🎛️

Per-Environment Tuning

Set different polling rates and thresholds for production (tight) vs. staging (relaxed). One account, different configs per host.

💡

Low Footprint Agent

The monitoring agent is lightweight — minimal CPU and memory usage. Configurable heartbeat intervals let you balance freshness vs. overhead.

Pros & Cons

Pros

  • ✓ Update agent settings without touching the server
  • ✓ Tune monitoring intensity per environment — less noise from dev/test
  • ✓ Low resource consumption on monitored hosts
  • ✓ Changes propagate immediately — no deploy pipeline needed
  • ✓ Useful for managing large fleets of servers efficiently

Cons & Considerations

  • ! Push requires the agent to be online and connected — offline agents receive config on next reconnect
  • ! Config is managed via dashboard only — no GitOps / IaC integration yet
  • ! Very high polling rates can increase load on the monitored host
